🧊 FridgeGeniusAI (SaveEat ✨)
AI·딥러닝 기반 스마트 냉장고 관리 & 음식 낭비 감소 Android 애플리케이션


FridgeGeniusAI는 단순히 냉장고 속 재료를 기록하는 애플리케이션이 아닙니다.  
본 프로젝트는 딥러닝과 생성형 AI 기술을 활용하여 사용자가 “무엇을 언제 소비해야 하는지”를 판단할 수 있도록 돕는 지능형 의사결정 지원 시스템을 목표로 설계되었습니다.

현대 사회에서 음식물 쓰레기는 환경적·경제적·윤리적 측면에서 중요한 문제로 대두되고 있습니다.  
그러나 기존의 냉장고 관리 애플리케이션은 대부분 다음과 같은 한계를 가지고 있습니다.

- 재료를 사용자가 직접 하나하나 입력해야 하는 번거로움
- 유통기한 정보는 제공되지만 소비 우선순위 판단은 사용자 책임
- “그래서 지금 무엇을 먹어야 하는가?”에 대한 AI 기반 판단 부재

FridgeGeniusAI는 이러한 문제를  
**‘AI가 판단을 제안하고, 사용자가 최종 결정을 내리는 구조’**로 재설계한 프로젝트입니다.


────────────────────────────────

문제 정의 – 기존 냉장고 관리 방식의 한계

기존 냉장고 관리 방식에서는 정보는 존재하지만 판단은 온전히 사용자에게 맡겨져 있습니다.  
유통기한을 확인할 수는 있으나, 어떤 재료가 더 급하게 소비되어야 하는지 직관적으로 알기 어렵습니다.  
이로 인해 소비 결정이 지연되고, 결과적으로 음식 낭비로 이어지는 문제가 발생합니다.

또한 텍스트 기반 입력 방식은 번거롭고 누락되기 쉬워 사용 지속성을 떨어뜨립니다.  
AI를 활용한다고 하더라도 단순 레시피 추천 수준에 머무르는 경우가 많아,  
실제 행동 변화로 이어지지 않는 “알지만 행동하지 않는 문제”가 반복됩니다.


────────────────────────────────

프로젝트 목표

FridgeGeniusAI는 다음 다섯 가지 핵심 목표를 중심으로 설계되었습니다.

1. 유통기한 정보를 기반으로 재료의 소비 위험도를 정량화
2. 지금 당장 소비해야 할 재료를 AI가 자동으로 판단
3. 실제 조리가 가능한 레시피를 생성형 AI로 제공
4. 딥러닝 기반 이미지 인식을 통한 재료 입력 자동화
5. 사용자의 행동 변화를 유도하는 UX 설계


────────────────────────────────

시스템 전체 구조 및 흐름

사용자는 텍스트 입력 또는 사진 촬영을 통해 재료를 등록합니다.  
사진 입력의 경우 Google ML Kit 기반 이미지 인식 모델이 재료 후보와 신뢰도를 제공합니다.

입력된 재료 정보는 Room Database에 저장되며,  
각 재료의 유통기한을 기준으로 위험도 점수를 계산하는 모델이 적용됩니다.

이후 AI 판단 레이어에서는  
위험도 점수가 높은 재료를 자동으로 추출하고,  
해당 재료를 중심으로 GPT 기반 레시피를 생성합니다.

최종적으로 결과는 UX 화면을 통해  
홈 미션, 레시피 추천, 인사이트 분석 형태로 사용자에게 제공됩니다.

👉 AI 판단은 단일 기능이 아니라,  
전체 UX 흐름을 관통하는 핵심 레이어로 작동합니다.


────────────────────────────────

핵심 AI 및 딥러닝 기능 설명

유통기한 기반 위험도 점수 모델은 각 재료의 남은 유통기한을 기준으로  
0에서 100 사이의 위험도 점수를 계산합니다.  
이 점수는 단순한 선형 계산이 아닌 구간별 가중치 모델을 적용하여  
실제 소비 긴급도를 반영합니다.

- 0~39점: 비교적 여유 있는 상태
- 40~69점: 주의가 필요한 상태
- 70~100점: 즉시 소비가 권장되는 상태

이 위험도 점수는 모든 AI 판단의 기준 지표로 활용됩니다.

위험도 점수를 기반으로 AI는  
“오늘 반드시 소비해야 할 재료 TOP 3”를 자동으로 추출합니다.  
사용자가 별도의 설정을 하지 않아도 소비 우선순위를 직관적으로 확인할 수 있어  
결정 피로도를 크게 줄여줍니다.

레시피 추천 기능은 OpenAI GPT API를 활용하여 구현되었습니다.  
재료 상황을 인식하는 프롬프트를 설계하여  
실제 조리가 가능한 레시피만 생성하도록 제한하였습니다.

AI가 생성하는 레시피에는  
레시피 이름, 예상 조리 시간, 난이도, 사용 재료, 조리 단계,  
추가 팁 및 대체 재료 정보가 포함됩니다.  
이는 단순한 텍스트 생성이 아닌,  
사용자의 실제 행동을 유도하는 AI 출력을 목표로 합니다.

사진 기반 재료 인식 기능은 Google ML Kit Image Labeling을 사용합니다.  
온디바이스 딥러닝 모델을 통해 재료 후보와 신뢰도를 제공하며,  
오인식 가능성을 고려하여 사용자가 직접 선택·확정하는 UX 구조를 적용했습니다.

이를 통해 AI 판단과 사용자 통제 사이의 균형을 유지합니다.

또한 FridgeGeniusAI는 사용 패턴 분석과 인사이트 제공 기능을 포함합니다.  
AI 기반 음식 낭비 감소 지표, 예상 절약 비용,  
위험 재료 관리 성과를 시각적으로 제공하여  
사용자가 “이 앱을 사용하면서 어떤 변화가 있었는지”를 인식할 수 있도록 설계되었습니다.


────────────────────────────────

UI / UX 설계 철학

FridgeGeniusAI의 UX는 다음 원칙을 따릅니다.

- AI는 앞에 나서지 않고 판단을 제안하는 역할
- 최종 결정은 항상 사용자에게 귀속
- 색상과 아이콘을 활용한 직관적인 위험도 전달
- 버튼 최소화 및 행동 유도 중심 UI 구성


────────────────────────────────

사용 기술 스택

Android: Kotlin, Jetpack Compose, Material Design 3  
AI / ML: OpenAI API (GPT), Google ML Kit Image Labeling  
Data: Room Database, Flow / StateFlow  
Architecture: MVVM, Hilt, Coroutines


────────────────────────────────

프로젝트 차별점

FridgeGeniusAI는 AI 기능이 부가 요소가 아닌 프로젝트의 중심에 위치합니다.  
딥러닝 결과를 UX로 보완하여 실제 사용 환경을 고려했으며,  
평가용 데모가 아닌 서비스 지향적 구조로 설계되었습니다.

또한 향후 알림, 예측 모델 고도화, Gemini API 연동 등  
확장 가능한 구조를 고려해 설계되었습니다.


────────────────────────────────

결론

FridgeGeniusAI는  
AI와 딥러닝 기술을 실생활 문제 해결에 적용한 실무형 프로젝트입니다.

본 프로젝트는 단순히 기술을 나열하는 데 그치지 않고,  
왜 AI가 필요한지,  
AI가 어떤 판단을 내리는지,  
그 판단이 어떻게 사용자 행동을 바꾸는지를 중심으로 설계되었습니다.

FridgeGeniusAI는  
기술을 보여주기 위한 프로젝트가 아니라,  
문제를 해결하기 위해 AI를 사용하는 프로젝트입니다.
